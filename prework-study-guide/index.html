HTML

  • The head element contains information(metadata) about the webpage that helps the browser render it correctly
  • The body element represents the visible content shown to the user
  • !DOCTYPE is the decleration at the top that tells the browser what document type should be expected when opened
  • Style element defines the CSS styles associated with a website
  • Base ewlements defines the base URL for a webpage
  • Link element connects an external resource to the HTML document
  • Meta element defines metadata such as the character set, description, keyowrds, author and viewpoint
  • Script element embeds client-side scripts in the HTML
  • h1-main header, h2-second teir of headings
  • ul-unordered list, li-list item

CSS

  • Cascading Styl Sheets(CSS)- a programming language used by developers to define how webpage content is presented to users
  • Inline CSS styles apply to a specific HTML tag using a style attribute with a CSS rule to style a particular page element only
  • Ex: (h1) style=color:red:>Prework Study Guide(/h1)
  • Internal Style sheet contains CSS rules for the webpage in the head section of the HTML file
  • Ex: (head) Line 1 (style) Line 2 hi { Line 3 color:red; Line 4 } Line 5 (/style) Line 6 (/head) Line 7
  • External CSS style is the most common eway to style a webpage because it allows develpers to keep all their CSS rules in a seperate file which makes desiogn changes easier
  • when using an external style sheet we link the file using a (link) element placed in the HTML's (head)
  • (head) Line 1 (link rel="stylesheet" href"./assets/style.css" /) Line 2 (/head) Line 3
  • CSS rule- a selector defines the element or attributes to which the rules, or declarations, will apply
  • Declerations contain two important components: the CSS property we want to apply and the value of the property
  • Margin property indicates how much space we want around the outside of an element. The margin is not visible on the page, it merely adds a buffer
  • The padding property adds space around the content inside of an element

Git

  • Repository(Repo) in Github stores files for projects and the version history for those files
  • Github issues usually include a title and description that has user stories and acceptance criteria top describe the the task that needs to be completed
  • GitHub flow- Create a feature branch, make changes, commit and push your changes, create a pull request, address/review comments, merge pull request
  • The git command must precede every Git statement
  • Git status: checks what branch we are currently on
  • Git checkout: moves the working branch to anew branch
  • Git checkout -b branch-name: creates a new branch and switches to it
  • It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch
  • Git add -A: commit all files in the working branch
  • Git pull origin main: syncs your local and remote main branches

JavaScript

  • Tool that takes a static HTML/CSS webpage and makes it interactive
  • Create file with name script.js to work on then add to HTML using (script) element under (footer)
  • Use (console.log) to log variable to the web console in browser
  • Variables: named containers that store data in our code.
  • Control flow: the order in which a computer executes code in a script
  • Array: a single variable that ius used to hold a group of data. Typically used to hold data that is related in some way
  • Function: set of instructions that tells a computer how to perform a certain task (do not automatically execute when the Js file is run. must call a function by its name to execute it)
